## 1. App Store Connect での操作フロー

### 1.1 新バージョン作成

1. App Store Connect → 「マイ App」→ おてつだいコイン → 左メニュー「App Store」
2. 「+ バージョンまたはプラットフォーム」 → iOS App → バージョン番号 `1.1.0` を入力

### 1.2 ビルド選択

「ビルド」セクションで Build 36 を選択 (TestFlight で配信済みのもの)。

### 1.5 App Privacy (App プライバシー)

Apple Developer Program License Agreement / App Store Review Guideline Section 5.1 の関連。

- **収集するデータ**: 「データを収集していません」ではなく、AdMob 経由で **デバイス ID (識別子)** が広告配信のために共有される旨を申告する必要があります。
- ATT (#22) で `npa=1` (Non-personalized) 運用中なので、「**ユーザーまたはデバイスに関連付けられた識別子**」のうち「**広告 (Non-personalized)**」目的に絞った形で申告。
- 既に **Privacy Manifest (PrivacyInfo.xcprivacy)** をリポジトリに追加済み (#22 フォローアップ) なので、その内容と整合性を取る形でチェック。

### 1.6 価格・配信地域

v1.0 から変更がなければ「変更なし」。

### 1.7 ATT / IDFA 申告

「App はデバイスの広告識別子 (IDFA) を使用しますか？」の質問に **「いいえ」** で回答。Non-personalized 運用なので IDFA を要求していません。

### 1.8 エクスポートコンプライアンス

`Info.plist` の `ITSAppUsesNonExemptEncryption = NO` を設定済み (project.pbxproj の `INFOPLIST_KEY_ITSAppUsesNonExemptEncryption = NO`)。ASC でも改めて「暗号化を使用していない」を選択。

### 1.9 「審査へ提出」

すべて入力したら「審査へ提出」ボタン。手動リリース or 自動リリースを選択。

## 4. よくある reject 理由と対策

### 4.1 「プライバシーポリシーが App の挙動と一致しない」
- 現状の `PRIVACY_POLICY.md` は AdMob 導入前の文言で、「広告の表示は行わない」と書かれており実装と矛盾しています。
- **対策**: 提出前に [#39](https://github.com/es0612/OtetsudaiCoin/issues/39) で本文を更新し、`docs/privacy-policy.html` も同期。

### 4.2 「Kids カテゴリ要件を満たしていない」
- もしカテゴリを「キッズ」にしている場合、Apple は AdMob 広告に **追加の制約** (DEM 認証など) を課します。
- **対策**: カテゴリは「ライフスタイル」または「教育」を選択し、年齢レーティングを「4+」または「9+」で運用するのが無難。「キッズ」カテゴリは選ばない。

### 4.3 「IDFA を使うのに ATT 申告がない」
- 今回 Non-personalized (`npa=1`) で IDFA を使っていないので、ATT 質問には **「いいえ」** で回答。
- **対策**: ATT 質問への回答が「いいえ」になっていることを再確認。`Info.plist` に `NSUserTrackingUsageDescription` を**含めていないこと** も確認 (含めると「使っている」と見なされる)。

### 4.4 「スクリーンショットがアプリの実態を反映していない」
- v1.0 のスクショで v1.1.0 を申請しても通常は通りますが、目立つ UI 差分があると指摘されることがあります。
- **対策**: 履歴画面・設定画面は撮り直し。

### 4.5 「app-ads.txt 検証エラー」
- これは Apple 審査の指摘ではなく **AdMob 側** の警告。検証が「承認済み」になっていなくても審査自体は通りますが、広告配信レートが下がります。
- **対策**: GitHub Pages 公開後 AdMob 管理画面で再検証を促す。

## 5. 審査開始後の対応

### 5.1 標準審査期間
- 通常 1〜3 日。AdMob 利用アプリは初回審査でやや長くなる傾向 (今回は 2 回目の審査なので 1〜2 日見込み)。

### 5.2 リジェクトされた場合
1. Resolution Center で指摘内容を確認
2. 該当する `RELEASE_v1.1.0.md` セクションまたは [#39] を参照して修正
3. ビルド再 Archive が必要なら build number を 37 にインクリメントして再アップロード
4. 文言だけ修正なら ASC のメタデータ修正で再提出

### 5.3 承認後
- 自動リリースなら即時 App Store に公開
- 手動リリースなら ASC の「リリース」ボタンを押す
- 公開後、AdMob 管理画面で広告配信が始まることを確認
